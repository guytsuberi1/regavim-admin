# תפעול — ישיבת רגבים בנימין (regavim-admin)

אפליקציית הניהול האדמיניסטרטיבי של גיא צוברי: מחזור דוחות השכר החודשי, ניהול משימות,
ניהול פרויקטים ותקציבים, ופורטל דיווח לעובדים.
חלק ממשפחת האפליקציות של רגבים בנימין (regavim-budget · regavim-dorm · regavim-agriculture).

🔗 **אפליקציה**: https://guytsuberi1.github.io/regavim-admin/
🔗 **פורטל דיווח לעובדים** (פתוח): https://guytsuberi1.github.io/regavim-admin/report.html

## טכנולוגיה

- **Frontend**: Vanilla HTML/CSS/JS, ללא build. `admin-site/` הוא האתר כולו.
- **Backend**: Supabase (אותו פרויקט כמו שאר האפליקציות) — PostgreSQL + Auth + Realtime + Storage.
- **פריסה**: GitHub Actions מפרסם את `admin-site/` לענף `gh-pages` בכל push ל-main.
- **עיצוב**: styles.css משותף עם regavim-dorm (RTL, Rubik, סרגל צד).

## גיליונות

הניווט דו-שכבתי: **גיליון** בסרגל הצד → **תת-טאבים** בתוך התוכן.

| גיליון | תוכן |
|---|---|
| 💼 **דוחות שכר** | לוח שכר חודשי · תור אישורים · מרכז למידה · מילוי מקום · היעדרויות וגמולים · חבילת PDF |
| ✅ **ניהול משימות** | טבלה + קנבן, עדיפויות ותאריכי יעד, משימות קבועות שמתחדשות אוטומטית |
| 🏗️ **ניהול פרויקטים** | כרטיס לכל פרויקט עם פס תקציב/נוצל/מאזן אוטומטי + תת-משימות (גרירה, כיווץ) |
| 🗂️ **נתוני בסיס** | מצבת עובדים, ייבוא מאקסל, תגיות והרשאות |
| ⚙️ **הגדרות** | תעריפים, חתימת מנהל, גיבוי ושחזור |

## מבנה הנתונים (Supabase)

| טבלה/משאב | תוכן | הרשאות |
|---|---|---|
| `admin_state` | מצב האפליקציה בשורות JSONB: `core` (מצבת+הגדרות), `lc:YYYY-MM`, `sub:YYYY-MM`, `abs:YYYY-MM`, `pstat:YYYY-MM`, `tasks`, `projects`, `portal` | משתמשים מחוברים: הכל; אנונימי: קריאת `portal` בלבד |
| `admin_submissions` | דיווחי עובדים מהפורטל (pending/approved/rejected) | אנונימי: INSERT בלבד; מחוברים: הכל |
| Storage `admin-approvals` | קבצי אישורים (מחלה/3010) | אנונימי: העלאה; מחוברים: צפייה |

ההקמה: הרצת `supabase/schema.sql` ב-SQL Editor (חד-פעמי, בטוח להרצה חוזרת).

## דפים

- `index.html` — האפליקציה (דורש התחברות).
- `report.html` — פורטל דיווח פתוח לעובדים (ללא התחברות): היעדרות + העלאת אישור,
  נסיעות, טיול. כל דיווח נכנס לתור האישורים לאישור גיא.

## תפקידים

- **admin** (גיא, לפי אימייל ב-`ADMIN_EMAILS` או תפקיד במצבת) — הכל.
- **secretary** (המזכירה) — מסך מרכז למידה בלבד. זו גם ברירת המחדל למשתמש לא מוכר.

## מוסכמות פיתוח

- אחרי שינוי CSS/JS — להעלות את `?v=N` בכל התגיות ב-`index.html` (עקיפת cache).
- תעריפים (שעת תגבור, ק"מ) ורשימות (תחומים, אחראים, קבלנים) — במסך ההגדרות, לא בקוד.
- הריפו קיים גם ב-OneDrive במחשבי גיא וגם בסביבות ענן —
  **`git pull` לפני עבודה, `git push` בסוף**, אחרת העותקים מתפצלים.

הקשר מלא לסשן פיתוח חדש (כולל החלטות תוכן שכבר סוכמו והגיליונות שנותרו) — ב-[CLAUDE.md](CLAUDE.md).
